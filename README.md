# Lab 6: Next.js vs Remix

## What's this lab about?

You already know React — but React on its own is just a **library**. It gives you the tools to build UI components, but it doesn't tell you how to handle routing, data fetching, or how to structure a full application.

**Next.js and Remix are frameworks built on top of React.** Think of React as the bricks, and these frameworks as the blueprint — you're still writing the same React code you know, but inside a structure that solves the bigger architectural problems for you.

This lab is a chance to explore one of these frameworks and get a feel for how the real-world React ecosystem works beyond just React itself.

---

## Choose your path

### Option 1 — Learn a little Next.js

**What is Next.js?**
Next.js is a React framework built by Vercel. It's one of the most widely used frameworks in the industry. It handles routing, server-side rendering, and data fetching on top of React, and gives you a lot of flexibility in how you build your app.

**Tutorial:**
[Next.js Tutorial for Beginners | Nextjs 13 (App Router) with TypeScript](https://www.youtube.com/watch?v=ZjAqacIC_3c)

> ⚠️ The tutorial asks you to install a specific version of Next.js. **Ignore that.** Instead, run:
> ```bash
> npx create-next-app@latest
> ```
> The UI may look slightly different from the tutorial since it was recorded in 2023, but the core concepts are the same.

### Option 2 — Learn a little Remix

**What is Remix?**
Remix is a newer React framework created by the team behind React Router. It takes a more opinionated approach — it's built closely around web platform standards like native forms and the browser's built-in fetch. If Next.js is flexible and feature-rich, Remix is focused and philosophically pure. Neither is better — they just make different tradeoffs.

**Tutorial:**
[Remix Tutorial For Beginners - Learn Remix In 1 Hour](https://www.youtube.com/watch?v=d_BhzHVV4aQ)

## A note on both frameworks

Both Next.js and Remix are used in real production applications. As you explore, pay attention to:

- How routing works
- How data gets loaded into pages
- What feels familiar from React, and what feels new

You don't need to finish the tutorial — the goal is exposure and curiosity.
